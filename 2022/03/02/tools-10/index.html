什么是 protobuf

protobuf 全称为 Protocol Buffers，是一种轻便高效的结构化数据存储格式，可以用于结构化数据的序列化，很适合做数据存储或 RPC 数据交换格式。protobuf 由 Google 公司开发，有着语言无关，平台无关，可拓展性强的特性，相比于 XML，JSON 这些数据交换格式，protobuf 更是有着更快，更小，更简洁的特点。

protobuf 可以将数据存储为序列化的二进制格式，采用一种压缩的方式，这也是为什么在网络传输的时候其受到很大欢迎的原因，毕竟又小又快嘛！

实际使用上，我们可以将 protobuf 当成一种语言，通过其语法定义数据的格式，然后便可以通过其编译器生成相关高级语言的接口代码，这样便可以简单的用我们熟悉的高级语言（如 Java，C++）来编写读写数据的接口。protobuf 目前支持Java、Python、Objective-C 和 C++ 中生成的代码。在最新的 proto3 语言版本中，也可以使用 Kotlin，Dart，Go，Ruby 和 C#，以及更多的语言。

protobuf 语法

正如上述所说，protobuf 可以被当成一门简单的语言，语言自然有语法了，而 protobuf 现在支持两种语法（proto2 和 proto3），下述则以 proto3 为例概述 protobuf 的语法。protobuf 的编译器名字是 protoc，其接收以 .proto 为后缀的文件作为输入，生成对应的高级语言源文件，比如在 C++ 中，protoc 会生成两个以 .pb.h 和 .pb.cc 为后缀的 C++ 源文件。

Tips

关于字段号（Field Numbers）:

  • 一旦定义好就不应该在后续的扩展中更改，保持 message 的兼容性。
  • 字段号 115 在存储时只占一个字节，而 162047 占两个字节，所以记住为你的 message 保留 1~15 这些字段号，将它们分配给那些很常用的数据字段。
  • 字段号19000~19999 不能使用，否则会报错，同样之前自己保留的字段号也不能再次使用。
  • 为了保持兼容性，在删除某个字段号的时候，记住保留删除的字段号，防止后续的二次使用破坏兼容性。
